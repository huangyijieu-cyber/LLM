# 参数高效微调 (Parameter-Efficient Fine-Tuning, PEFT)

参数高效微调是指固定大部分预训练模型参数, 仅微调少量或额外的模型参数, 从而大幅降低计算和存储成本, 同时实现与全参数微调相当的性能.

高效微调技术可以粗略分为以下三大类: 增加额外参数 (A), 选取一部分参数更新 (S), 引入重参数化 (R).

## 1. 增加额外参数 (A)

这一类方法的核心是**冻结原模型参数, 在模型中额外引入可训练模块**, 主要包括以下技术:

### 1.1 Adapter Tuning

Adapter Tuning 的核心思想是冻结原 PLM 的所有权重, 在模型的网络层中间插入一些小型的"适配器模块 (Adapters)".

#### 1.1.1 技术架构

Adapter Tuning 对输入文本序列 x 不做任何修改. 它直接修改大模型内部的架构, 在 Transformer 的每一层中增加额外的网络层 (Adapter), 以残差的形式直接加和到每一层中 (即 `h` -> `h + Adapter(h)`).

#### 1.1.2 具体实现

在标准 Transformer 层中, 主要包含两个核心子层: **MHA** 和 **FFN**, Adapter Tuning 会在每个子层之后各插入一个 Adapter 模块, Adapter 模型内部主要靠残差连接的方式参与计算 (假设输入 Adapter 的向量为 $h$):

1. 先将高维 $h$ 压缩到极低的维度: $h = W_{down} \cdot h + b_{down}$.
2. 再经过一个非线性激活函数 $h = f(h)$
3. 然后将其重新映射回高维: $h = W_{up} \cdot h + b_{up}$.
4. 残差连接得到最终输出: $Adapter(h) = h + W_{up} \cdot f(W_{down} \cdot h + b_{down}) + b_{up}$.

故 Adapter 计算方式可直观表示为:

$$
h' = h + Adapter(h)
$$

Adapter 作用于每一层的中间表示, 直接对隐藏状态进行变换.

#### 1.1.3 训练更新方式

冻结原 PLM 中的所有参数, 在反向传播中, 仅计算并更新所有 Adapter 内部的 $W_{down}$, $b_{down}$, $W_{up}$, $b_{up}$. 由于中间层维度远小于模型隐藏层维度, 这使得每层 Adapter 的参数量极小, 只需训练模型总参数的**0.5% - 8%**, 即可在多个下游任务上达到与全参数微调相当的表现.

由于 Adapter Tuning 在网络中强行插入了非线性激活函数 $f(\cdot)$, 这就决定了 Adapter 的权重矩阵 ($W_{down}$ 和 $W_{up}$) 在数学上无法与原 Transformer 的权重矩阵进行等效合并. 不同于其他方法的直接拼接, Adapter Tuning **在推理阶段不可避免地引入了额外显存占用**.



### 1.2 Prefix Tuning

Prefix Tuning 的核心思想是, 冻结 PLM 中的所有参数, 仅在输入 Token 之前构造一段任务相关的 **virtual tokens** 作为 Prefix, 来引导模型完成特定下游任务. 

#### 1.2.1 技术架构

针对不同模型结构, 需构造不同的 Prefix:
- **自回归架构模型 (Decoder-only)**: 在句子前面添加前缀(即 `[x; y]` -> `[Prefix; x; y]`), 合适的上文能在固定 PLM 的情况下引导生成下文. (如 GPT-3 的上下文学习)
- **Encoder-Decoder 架构模型**: 在 Encoder 和 Decoder 前都增加了前缀 (即 `[x; y]` -> `[Prefix0; x; Prefix1, y]`). Encoder 端增加前缀是为了引导输入部分的编码, Decoder 端增加前缀是为了引导后续 Token 的生成.

#### 1.2.2 具体实现

如果只把 Prefix 加在最底层的输入 Embedding 层, 那便成了 Prompt Tuning. 而 Prefix Tuning 直接在 Transformer 的每一层都硬编码了这段 Prefix. 具体实现方式是通过在 $K$ 和 $V$ 矩阵前拼接独立的可训练向量 $P_K$ 和 $P_V$:

$$
K' = [P_K; K], V' = [P_V; V]
$$

#### 1.2.3 训练更新方法

在更新 $P_K$ 和 $P_V$ 上, Prefix Tuning 使用了**重参数化技巧**. 这是因为 $P_K$ 和 $P_V$ 是高维度的连续浮点矩阵, 如果直接让优化器去更新这些高维参数, 训练过程会极度不稳定, 效果很差.

Prefix Tuning 引入了一个低维的 Embedding 矩阵 ($P'$) 和一个两层的 $MLP$ 层:

$$
P_K, P_V = MLP(P')
$$

训练阶段, 通过利用 $MLP$ 将小矩阵 $P'$ 映射成高维的 $P_K$, $P_V$, 并将它们拼接到冻结的 PLM 的每一层中, 以此来进行前向传播和反向传播, **只更新 $P'$ 和 $MLP$ 层的权重**. (**注意, $MLP$ 的权重是每层共享的**)

训练结束完成后, 我们只需将训练好的 $P'$ 通过 $MLP$ 重新映射回 $P_K$ 和 $P_V$ 即可, $MLP$ 层就可以丢弃了. 在推理时只需将 $P_K$ 和 $P_V$ 拼接到 $K$ 和 $V$ 上即可.

通过上述参数更新方式, Prefix Tuning 用不到 0.1% 的参数更新量, 能够实现媲美全参数微调的强大性能. 同时, 对多个不同的下游任务, 无需保存多个大模型的全部权重, 只需保存一份大模型权重 + 多个前缀向量即可完成任务模式的切换.

### 1.3 Prompt Tuning

Prompt Tuning 的核心思想是, 完全冻结 PLM 的参数, 仅在模型的 Embedding 拼接上一小段可训练的、连续的 **"软提示向量 (Soft Prompt)"**, 以此来激发大模型完成特定下游任务的能力.

#### 1.3.1 技术架构

该方法可以看作是 Prefix Tuning 的简化版本. 它给每个任务定义了自己的Prompt, 然后拼接到数据上作为输入, 但**只在输入层加入 prompt tokens** (即 `[x; y]` -> `[Soft_prompt; x; y]`). 

Prompt Tuning 在某种程度上类似于直接构造特定任务的 Prompt 进行微调. 但不同于构造 Prompt 需要人工设计适合下游任务的 Prompt (人为选择的往往不是最好的), Prompt Tuning 选择使用机器自己通过训练数据拟合出来的最佳 Embedding 向量.

#### 1.3.2 具体实现

不同于 Prefix Tuning 在 Transformer 的每一层都拼接前缀, Prompt Tuning 仅在最底层的 Embedding 层进行修改. 具体的视线方式是在文本转换成的向量序列 $E_x$ 前进行拼接 $P_e$:
$$
X_{input} = [P_e; E_x]
$$
随后送入模型中.

#### 1.3.3 训练更新方法

不同于 Prefix Tuning, 梯度在反向传播时, 会穿过整个被冻结的 PLM, 一直传导回最底层的输入端, 优化器**直接更新** $P_e$ 矩阵的权重.

在 Prefix Tuning 的参数更新方法中, 我们提到过, 直接对高维矩阵进行更新, 训练过程会极不稳定, 容易陷入局部最优. Prompt Tuning 采用直接更新成功的原因是什么呢? 
1. **初始化方法**: Prompt Tuning 没有采用随机初始化, 而是采用了真实的单词 Embedding 作为 $P_e$ 的初始值. 
2. **模型规模**: 当模型参数量较小时, Prompt Tuning 完全不如全参数微调. 但当模型规模达到百亿级别时, Prompt Tuning 的效果就会急剧上升, 甚至与全参数微调完全持平.

训练结束后, 我们将得到一个极小的, 仅针对特定任务的 $P_e$ 矩阵. 推理时, 我们只需查表获得输入文本 x 的 Embedding, 再将保存好的 $P_e$ 拼在前面即可.

### 1.4 P-Tuning

P-Tuning 的核心思想是, 用可学习的, 连续的 Virtual Token 来替代人工设计的自然语言模板.

#### 1.4.1 技术架构

Prompt Tuning 强制将 $P_e$ 放在句首, 而 P-Tuning 引入了"模式 (Pattern)"的概念, 允许将连续的 Virtual Token 穿插在文本的任何位置 (即 `[x; y]` -> `[prompt0; x; prompt1; y]`). 通过在输入序列中灵活地放置虚拟 Token, 构造出一个最适合当前任务的"填空题模板". 模型依然被冻结, 全靠这段连续模板去引导它输出预测结果.

#### 1.4.2 具体实现

和 Prompt Tuning 一样, P-Tuning 也仅仅作用于最底层的 Embedding 层. 但它们在连续向量的生成方式上有本质区别. 

在 Prompt Tuning 中, 输入的 Virtual Token 向量彼此之间是完全独立的, 各自随机初始化和更新. 而 P-Tuning 发现语言模型的 Embedding 空间本就高度离散, 如果让这些虚拟 Token 各自独立地在空间里搜索，优化起来极其困难. 他们认为一段好的 Prompt 应该是一段语义连贯的序列, 故提出了一个**序列模型 (Prompt Encoder)** 把这些 Virtual Token 先连接起来, 建立它们之间的上下文依赖关系.

对于生成的 Virtual Token 序列 $h_1, h_2, \dots h_k$, 按照模版进行拼接后得到: 
$$
X_{input} = [h_1, \dots h_i, E_x, h_{i+1}, \dots h_k, E_{mask}]
$$

#### 1.4.3 训练更新方式

P-Tuning 并没有直接去更新 Embedding 层那些 Virtual Token 的 Embedding 矩阵, 而是引入了一个额外的微型网络: **双向 LSTM (Bi-LSTM) + 两层多层感知机 (MLP)**.

假设我们需要 k 个 Virtual Token: $h_1, h_2, \dots h_k$. 前向传播时:
1. 先为 k 个位置生成一个初始低维的伪 Embedding.
2. 然后将它们输入 Bi-LSTM 中, 确保上下文关系.
3. LSTM 的输出再经过一个 MLP, 映射回 Embedding 层维度, 得到最终的连续软提示向量 $h_i$.
4. 然后将生成的 $h_i$ 与真实文本 x 的 Embedding 按照模版拼接到一起, 送入冻结的 PLM 中.

优化器只更新这个微型的 Prompt Encoder (Bi-LSTM + MLP) 的权重. 通过这个编码器作为"中间商"进行平滑过滤, 大大降低了优化的难度, 使得训练过程极其稳定.

训练结束后, 同 Prefix Tuning 一样, 将低维 Embedding 通过 Prompt Encoder 映射回 $h_i$ 后, 可将 Prompt Encoder 丢弃了.

### 1.5 P-Tuning v2

之前的 Prompt Tuning 和 P-Tuning 等方法存在两个主要的问题:
- 缺乏通用性.
   - 缺乏模型规模通用性: 当模型规模超过100亿个参数时, Prompt Tuning 可以与全量微调相媲美. 但对于较小模型, 性能很不理想.
   - 缺乏任务通用性: 在一些 NLU 基准测试中表现出优势, 但对序列标注等任务的有效性尚未得到验证.
- 缺少深度提示优化, 只出现在 Transformer 的第一层输入中, 可调参数太少, 影响太直接太弱.

故 P-Tuning v2 应运而生. 它的核心思想是, 彻底打破之前"只在输入层加提示"的局限, 借鉴 Prefix Tuning 的思路, 将连续的 Soft Prompt 渗透到 Transformer 的每一层中.

#### 1.5.1 技术架构

在 v1 中, P-Tuning 必须设计复杂的模版以激发模型理解能力. 而在 v2 中, 由于模型深层能力的释放, 剔除了这一繁琐的步骤, 回归了最简单的头部拼接 (即 `[x; y]` -> `[prompt; x; y]`).

#### 1.5.2 具体实现

P-Tuning v2 同 Prefix Tuning 一样, 选择将 prompt 向量拼接到每一层的 $K$, $V$ 矩阵前, 而不在 Embedding 层显式添加额外的 Virtual Token:

$$
K' = [P_K; K], V' = [P_V; V]
$$

同时在模型最顶层输出的 Prompt Token 位置上, 外接一个随机初始化的**线性分类头**.

#### 1.5.3 训练更新方式

P-Tuning v2 发现重参数化对模型性能提升效果有限, 于是放弃了繁琐的重参数化网络, 采用直接更新的方式. 在 Transformer 的每一层中, 都直接初始化一个相互独立的 Prompt 矩阵, 将其拼接到该层的 $K$, $V$ 矩阵上. 经过所有层后, 利用顶层的线性分类头输出结果计算 Loss. 然后对线性分类头和每一层的 Prompt 矩阵进行反向更新.

训练结束后, 我们得到了**一组适配于各个 Transformer 层的深层 Prompt 矩阵**, 以及一个顶层的分类头. 在推理时, 只需将 Prompt 矩阵插入 PLM 中, 最后通过分类头输出结果.

#### 1.5.4 与 Prefix Tuning 的差异

可以看出, P-Tuning v2 的实现细节几乎与 Prefix Tuning 一模一样. 可以看作是将文本生成的 Prefix Tuning 技术适配到 NLU 任务中, 然后做了一些改进：
1. **移除重参数化网络**.
2. **针对不同任务采用不同的提示长度**.
3. **引入多任务学习**.
4. **回归传统的分类标签范式, 而不是映射器**.

## 2. 部分参数更新 (S)
 
这一类的核心是**不引入新参数, 只更新原模型的一部分参数**, 主要包括以下技术:

### 2.1 Bitfit

Bitfit 是一种稀疏的微调方法, 它在训练时只更新**bias的参数或者部分bias参数**, 其余参数被冻结.

对于一个 Transformer 架构的模型来说, 涉及到的bias参数有 
- Attention 模块中计算 $Q$ $K$ $V$ 矩阵跟合并多个 Attention 结果时涉及到的 bias.
- MLP 层中的 bias.
- Layer Normalization 层的 bias.

BitFit 微调结果相对全量参数微调而言, 只更新极少量参数的情况下, 在多个数据集上都达到了不错的效果. 虽不及全量参数微调, 但是远超固定全部模型参数的 Frozen 方式.

## 3. 引入重参数化 (R)

这一类方法的核心思想是**不直接训练原权重，而是用一个结构化形式表示权重更新**, 核心技术包括:

### 3.1 LoRA

LoRA (LOW-RANK ADAPTATION OF LARGE LANGUAGE MODELS), 该方法的核心思想就是通过低秩分解来模拟参数的改变量, 从而以极小的参数量来实现大模型的间接训练. 这是目前最主流的 PEFT 技术.

#### 3.1.1 技术架构

在 Transformer 中，大量计算集中在线性映射 (如 Attention 中的 $W_q, W_k, W_v, W_o$ 以及 FFN 层), LoRA 的做法是: 对这些权重矩阵 $W \in \mathbb{R}^{d \times k}$, 不直接更新它, 而是引入一个低秩分解的增量 (即 `W` -> `W + ΔW`).

LoRA 并不改变原有模型的网络深度, 而是通过平行分支来学习下游任务所需的增量特征.

#### 3.1.2 具体实现

LoRA 通常被应用于注意力机制中的 **Query** ($W_q$) 和 **Value** ($W_v$) 投影矩阵上 (也可扩展至所有线性层).

假设某个被冻结的预训练权重矩阵为 $W_o$ (`[d, k]`), 输入向量为 $x$ (`[B, S, k]`), 原始计算方法为 $h = W_o \cdot x$. Lora 的计算方法为:

1. LoRA 不直接训练 $W_o$, 也不直接训练维度相同的 $ΔW$, 而是选择了两个低秩矩阵 (秩为 r, 通常取 4, 8 等极小值) 的乘积来表示 $ΔW$:
   - 降维矩阵: $A$ (`[r, k]`)
   - 升维矩阵: $B$ (`[d, r]`)
   - 故增量矩阵可表示为 $ΔW = B \cdot A$
2. 当 $x$ 到达该层时, 数据会分为两路:
   - 原始权重 (冻结): 计算 $W_ox$.
   - 增量权重 (可训练): 计算 $BAx$.
3. 将两条路径结果相加, 并在增量权重处乘以一个缩放因子, 即可得到 LoRA 层最终输出:
$$
h = W_ox + \frac{α}{r}BAx
$$
   其中 α 是一个常数超参数. 引入 $\frac{α}{r}$ 的目的是在改变秩 r 的大小时, 减少对学习率等超参数的重新调整需求.

#### 3.1.3 训练更新方式

$A, B$ 权重是靠两个线性层实现的, $A$ 将维度 k 映射到低秩 r 上, 再由 $B$ 将低秩 r 映射到维度 d 上.

这使得其反向传播时参数更新极为简单, 仅计算并更新 $A$ 和 $B$ 两个线性层的参数即可. 

但 $A$, $B$ 两个矩阵的初始化需要遵循一定的要求:
- $A$: 使用随机高斯分布进行初始化.
- $B$: 严格使用**全 0** 进行初始化.

这使得初始状态下 $ΔW = BA = 0$. 确保训练起点的稳定性.

在训练结束后, 只需将训练好的低秩矩阵相乘并按缩放因子缩放后, 直接加和到原始权重上, 即可得到一个新的合并后的权重矩阵 $W_{merged}$:
$$
W_{merged} = W_o + \frac{α}{r}BA
$$

然后将 LoRA 旁路丢弃即可. 至此, 模型在推理时的网络结构, 层数, 计算步骤与微调前完全一致, 实现了绝对的**零推理延迟**. 

### 3.2 AdaLoRA

AdaLoRA (Adaptive LoRA) 是对 LoRA 的改进, 其核心思想是**动态调整每层 LoRA 的低秩矩阵大小 r**, 而不是像原始 LoRA 那样对所有层使用固定秩, 从而在保证性能的同时进一步压缩参数量.

#### 3.2.1 技术架构

与 LoRA 类似, AdaLoRA 仍然作用于 Transformer 的关键线性层* (如 Attention 中的 $W_q, W_v$). 不同的是, AdaLoRA 为每个层自动学习一个稀疏的秩分布:
- 对**某些关键层**保留较高的秩以维持性能.
- 对**冗余层**使用低秩或直接裁剪部分通道以减少参数.

#### 3.2.2 具体实现

AdaLoRA 引入了一个**可训练的门控机制**来控制每层实际使用的秩:
$$
ΔW=B⋅(A⊙Gate)
$$
其中 $Gate \in [0,1]^{r}$ 是按**秩维度**作用的缩放向量, **即每个 Gate[i] 对应一个低秩通道, 对 $A[i, :]$ 和 $B[:, i]$ 整列 (或整行) 进行缩放. 这种设计保证了 AdaLoRA 的"自适应秩"特性, 而不仅仅是稀疏化矩阵元素. $Gate$ 初始化为全 1.

除此之外与 LoRA 的具体实现几乎一致.

#### 3.2.3 训练更新方式

训练时更新的参数包括：
- $A$ 和 $B$ 两个线性层参数.
- 每个低秩维度对应的门控权重 $Gate$.

训练结束后, 跟 LoRA 一样, 只需将训练好的 $A$ 和 $B$ 按门控稀疏裁剪计算增量矩阵 $ΔW$, 再加和到原始权重 $W$ 上即可.

### 3.3 QLoRA

QLoRA (Quantized LoRA) 是在 LoRA 基础上结合**低比特量化**的参数高效微调方法, 旨在在极大模型上进一步减少显存占用, 同时保持训练性能.

#### 3.3.1 技术架构

为降低训练时显存, QLoRA 将原始权重矩阵 $W \in \mathbb{R}^{d \times k}$ 量化为 4-bit 或 3-bit 表示. 保留高精度缩放因子 (scale) 来保证数值精度, 避免量化误差过大. 其余技术与 LoRA 几乎一致.

#### 3.3.2 具体实现

在训练阶段之前, 使用 **NF4（Normal Float 4-bit）量化**或其他对称/非对称量化方案对原始权重的激活值进行分布式归一化, 使低比特表示尽量保留浮点精度. 这一步可减少显存占用 2~4 倍.

除此之外与 LoRA 的具体实现几乎一致.

#### 3.3.3 训练更新方式

训练过程和 LoRA 基本一致. 但对增量矩阵 $ΔW$, 由于其为低秩矩阵, 参数量远小于原模型权重, 故对其**保持浮点计算**, 以确保微调的精度和梯度稳定性.
$$
h=Dequantize(W_{quant}​)⋅x+ΔW_{float}​⋅x
$$

训练结束后, 直接将 LoRA 增量矩阵 $ΔW_{float}$ 与量化矩阵 $W_{quant}$ 合并. 用于后续推理计算.
