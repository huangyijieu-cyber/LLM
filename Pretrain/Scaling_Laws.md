# Scaling Laws

Scaling Laws 的核心思想是通过在小规模模型上建立简单的预测定律, 以此来指导大规模模型的工程决策和优化.

## 1. Data Scaling Laws (数据扩展定律)

数据扩展定律描述了在模型参数不变的情况下，增加训练数据量对模型性能的提升规律.

在双对数坐标下, 数据集规模 (Dataset_size) 与测试损失 (Test Loss) 呈现线性关系: $L(N) \propto N^{-\alpha}$ (幂律衰减). 这意味着数据越多, 模型损失下降越多, 但存在边际收益递减效益.

数据扩展定律有以下应用:
1. **数据集的组成成分主要影响 Scaling Law 的截距而非斜率**. 这意味着不需要在大规模模型上测试哪种数据配比最好, 只需在更小的模型上测试, 即可得到在大模型表现最好的数据集配比.
2. **多轮训练**, 研究表明, 重复数据存在边际递减效益, 引入了有效数据量的概念.
3. 是重复高质量数据还是引入新数据好, 可以通过拟合 Scaling Law 寻找权衡点.

## 2. Model Scaling Laws (模型扩展定律)

在固定训练数据量下, 增加模型参数大小会使模型性能按幂律规律提升:

### 2.1 超参数选择

研究表明, 在固定参数量下, 不同宽高比的模型 Loss 均呈现 U 形. 只要宽高比在一定范围内 (如 4 到 16 甚至 100), 性能都接近最优.

在做 Scaling 分析时, Embedding 参数不遵循同样的扩展规律. 因此计算总参数时, 不要计算 Embedding 层的参数. 只计算非 Embedding 参数能得到更干净的幂律.

### 2.2 不可直接迁移的超参数

大多超参数可以在小规模调整后直接迁移, 但 Batch Size 和 Leaning Rate 与规模紧密相关.

#### 2.2.1 Batch Size

临界 Batch Size (通常认为是最优 Batch Size) 是并行化收益递减的拐点:
- 当 Batch Size < 临界值: 增加 Batch Size 相当于增加梯度步数 (完美扩展, 时间效率高).
- 当 Batch Size > 临界值: 增加 Batch Size 不再显著减少所需的梯度步数 (数据效率低), 收益受限于优化的曲率.

原因:
- 在训练早期, 梯度大, 方向明确, 受限于曲率, Batch Size 不需要太大.
- 训练后期, Loss 变小, 梯度噪声占主导, 为获得准确下降方向, 需要更大的 Batch Size 拟合 来抵消噪声影响. 故目标 Loss 越低, 临界 Batch Size 越大.

**故随着 Loss 下降, 应当增大 Batch Size.**

**临界 Batch Size 主要随训练数据量的幂律增长**, 而对于固定数据量, 增大模型参数量并不会显著提高最优 Batch Size.

### 2.2.2 Learning Rate

**通常认为最优学习率随模型宽度 $W$ 衰减**, 经验法则是 $LR \propto 1/W$ 或 $1/\sqrt{W}$. 需要对不同规模的模型寻找最优 Learning Rate, 然后拟合规律进行外推.

存在一种参数化方法: **[MuP (Maximal Update Parametrization)](./MuP.md)**, 使得最优学习率在不同模型规模下保持稳定. 使得在小模型上测试出的最佳 Learning Rate 可直接用于大模型, 无需外推.

### 2.3 下游任务的非线性

Scaling Laws 对 Log Loss (Perplexity) 的预测极其精准, 但对于下游任务, 由于指标的离散性或涌现现象, 性能随规模的变化可能不是线性的, 甚至会出现不可预测的突变.

## 3. Chinchilla 定律

给定固定的计算预算 (FLOPs), 我们应该把资源花在更大的模型上, 还是更多的数据上? 即参数量 N 和数据量 D 预算的取舍.

Chinchilla 论文（Hoffmann et al., 2022）通过大量实验发现，在计算预算（FLOPs）最优分配下，模型参数量 N 和训练数据量 D（以 token 数计）的最优缩放关系满足：$N_{opt}$ ∝ $C^a$ 和 $D_{opt}$ ∝ $C^b$. 其中 a ≈ b ≈ 0.5. 这意味着参数量和数据量应该按相等的比例增加.

由 a ≈ b ≈ 0.5 可以推导出最优状态下模型参数量 N 与训练 token 数 D 满足线性关系： $D_{opt} / N_{opt} ≈ 20$, 即每增加一个参数, 需要增加 20 个 Token 的数据量来训练.

## 4. 推理最优扩展

Chinchilla 关注的是"训练最优"——即用最少的钱训练出最好的模型. 但在现实中, 大模型是产品. 我们不仅关心训练成本, 更关心推理成本.

由于推理是长期的运营成本, 现在的趋势是 Over-training (过度训练).

## 5. 局限性

Scaling Laws 暗示了"大就是好", 但在某些特定任务上, 模型越大表现反而越差, 这就是 Inverse Scaling.

Scaling Laws 的局限性主要在于在于其预测目标的高精确性仅局限于预训练阶段的单一 Next-Word 预测任务的交叉熵损失, 而无法直接等效为模型在具体下游任务中的表现.

在某些任务中, 随着模型规模增加, Loss 不降反升, 或者任务指标变差. 这通常发生在任务目标与预训练数据中学习到的分布冲突时, 例如某个任务要求模型避免复制, 而大模型相比于小模型更强的记忆与复制的能力就会成为任务的障碍. 而小模型能力弱, 反而更符合任务的要求.
